# Purpose

Detect and explain infrastructure, identity, configuration, and relationship changes over time.

Drift Analysis transforms snapshots into historical intelligence.

---

# Definition

Drift is any meaningful change between two points in time.

Examples:

New Asset

Removed Asset

Configuration Change

Permission Change

Exposure Change

Dependency Change

Identity Change
