# Data Visualization Standards

Version: 1.0

---

# Philosophy

Visualizations must prioritize clarity over aesthetics.

---

# Approved Chart Types

Line Charts

Area Charts

Bar Charts

Stacked Bars

Tables

Heatmaps

Node Graphs

Timelines

Treemaps

---

# Avoid

3D Charts

Pie Charts

Donut Overuse

Decorative Graphics

---

# Risk Visualizations

Use:

Heatmaps

Trend Lines

Distribution Bars

---

# Timeline Visualizations

Use:

Event Streams

Change Frequency Graphs

Drift Charts

---

# Graph Visualization Rules

Maximum visible nodes:

500

Beyond this:

Use clustering.

---

# Color Usage

Color represents:

Severity

Confidence

Status

Category

Never decoration.

---

# Dashboard Standards

Maximum:

8 visualizations per screen.

Avoid information overload.

---

# Tooltips

Required for:

Scores

Relationships

Confidence

Predictions

---

# Goal

Allow rapid understanding of complex infrastructure.
